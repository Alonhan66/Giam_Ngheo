## Route line appearance

Added ability to customize route line width and color. You can colorize route by selecting a custom color or by <Translate android="false" id="altitude"/>, <Translate android="false" id="gpx_slope"/>, <Translate android="false" id="routeInfo_roadClass_name"/>, <Translate android="false" id="routeInfo_surface_name"/>, <Translate android="false" id="routeInfo_smoothness_name"/>, <Translate android="false" id="routeInfo_tracktype_name"/>

You can find the customise line feature here: 
_OsmAnd Menu -> Settings -> Selected Profile -> Navigation settings -> Customomise route line_ 
or 
_[Navigation button](https://osmand.net/docs/user/widgets/map-buttons#directions) -> Options -> Customise route line_

## New map style

"Mapnik" map style was replaced with the new ["OSM-Carto" style](https://osmand.net/docs/user/map/vector-maps#default-map-styles). It was done because at the time when "Mapnik" was added to OsmAnd it was identical to map style on [OpenStreetMap.org](https://openstreetmap.org) but we didn't have recourse to keep it up to date, so over time, it was lost the similarity with OSM map style.

If you use "Mapnik" it will be available after the update but as a custom map style. After fresh or new installation you will need to manually import it to use. You can do it by downloading it from GitHub.

## Added copy actions to the "Share" sheet

Now, when you click ([long, short tap](https://osmand.net/docs/user/map/map-context-menu#context-menu)) to any point object on the map (POI, favorite, waypoint..) and choose ["Share" button](https://osmand.net/docs/user/map/map-context-menu#share) on the Map Context menu, you find adding copy actions. Chosen info will be copied to the buffer:

![Share sheet](./share-sheet_copy-actions.png)

* **Copy address** - allows to copy a point address to the bufer. Example from the image: _Kraków_. 
* **Copy POI Name** - allows to copy a point name to the bufer. Example from the image: _Kraków_. 
* **Copy coordinates** - allows to copy point coordinates to the bufer. Example from the image: _47.25963° N, 11.39645° E_.
* **geo:** - allows to copy a point geo tag to the bufer. Example from the image: _geo:47.25963,11.39645?z=19_.
